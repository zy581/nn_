# 车队协同自适应巡航与安全车距控制

本项目实现了一个面向智能网联汽车的车队协同自适应巡航控制演示模块。车辆队列在高速或城市快速路中行驶时，前车制动会沿着队列向后传播，如果后车只能依赖本车雷达测距进行普通 ACC 控制，速度扰动可能逐渐放大，导致后车频繁急刹、车距波动变大甚至产生追尾风险。本项目在普通 ACC 基线策略基础上，引入带通信延迟的 CACC 前馈控制，使后车能够提前感知头车加速度变化，从而更平稳地保持安全车距。

## 功能内容

- 构建 5 辆车组成的车队跟驰场景，头车包含减速、加速和二次扰动过程。
- 实现普通 ACC 基线控制，只依赖前车距离误差和相对速度。
- 实现 CACC 协同控制，在 ACC 反馈项上加入头车加速度前馈。
- 模拟 0.35 s 的 V2X 通信延迟，使控制过程更贴近真实车联网系统。
- 设置两段通信丢包窗口，观察控制器在通信不稳定时的安全性。
- 计算每辆跟驰车的实际车距、期望车距和车距误差。
- 比较同步队列中的最小安全车距、平均车距误差、最大车距误差、速度扰动放大率和舒适性代价。
- 生成速度曲线、车距误差热力图、指标对比图和时空轨迹图。

## 运行方法

在本目录执行：

```bash
python main.py
```

或指定输出目录：

```bash
python main.py --output assets
```

运行后会生成：

- `speed_profiles.png`：头车、普通 ACC 最后一辆车、CACC 最后一辆车的速度扰动传播对比。
- `cacc_gap_error_heatmap.png`：CACC 控制下各跟驰车辆的车距误差热力图。
- `metric_comparison.png`：普通 ACC 和 CACC 的核心指标对比。
- `space_time_trajectories.png`：CACC 车队时空轨迹，并标出通信丢包窗口。
- `cacc_gap_errors.csv`：各时刻各跟驰车的车距误差。
- `metrics.json`：实验指标和优化百分比。

## 项目意义

这个模块体现了从“单车智能”到“车路协同/车车协同”的优化思路。普通 ACC 只能看到最近前车，而 CACC 利用 V2X 信息获得头车加速度，使车队能更早响应扰动。即使存在通信延迟和短时丢包，控制器仍能通过反馈项维持安全车距，并在通信恢复后重新降低队列扰动。

该项目可以继续扩展到更多车辆、更复杂的通信故障、道路坡度、能耗模型、混合人工驾驶车辆和多车道编队换道等方向。

## 测试

```bash
python -m pytest tests
```

测试内容包括：

- CACC 是否降低平均车距误差和最大车距误差。
- 仿真过程中是否保持正向安全车距。
- 主程序是否能生成指标文件和全部可视化图片。
