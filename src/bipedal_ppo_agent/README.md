# Bipedal Walker PPO 训练项目

本项目使用**近端策略优化（PPO）算法**，在**双足行走机器人（Bipedal Walker）** 环境中训练强化学习智能体。该环境模拟了一个由4个关节、两条腿组成的双足机器人，其目标是在复杂地形中稳定行进。本项目同时实现了标准模式和硬核模式两种训练场景。

## 0. 关于Bipedal Walker环境
Bipedal Walker 是基于 Box2D 物理引擎开发的强化学习环境，模拟双足机器人在不同地形中的行走任务。智能体需要在应对地形和障碍的同时，保持平衡、协调动作并向前移动。

- **观测空间**：包含24个连续值，涵盖机器人机身角度、速度、关节角度及激光雷达（LIDAR）读数。
- **动作空间**：4个连续值，用于控制髋关节和膝关节的扭矩输出。
- **奖励机制**：向前移动获得正奖励，关节扭矩过大或机器人摔倒会受到负惩罚。
- **终止条件**：机器人摔倒或超出最大步数限制（标准模式为1600步，硬核模式为2000步）。

## 1. 项目结构
- **`main.py`**：主程序文件，包含标准模式和硬核模式的训练主循环。
- **`env_utils.py`**：环境工具脚本，用于配置Bipedal Walker环境，支持帧堆叠、视频录制、奖励归一化等扩展功能。
- **`logs/`**：训练日志文件存储目录。
- **`models/`**：训练完成的PPO模型保存目录。
- **`videos/`**：若启用视频录制，训练过程的视频文件将保存在此目录。

---

## 2. 训练流程
### 标准模式
- **训练步数**：100万步
- **环境**：标准 `BipedalWalker-v3` 环境
- **训练优化技术**：并行向量化环境、奖励归一化、帧堆叠、视频录制
- **模型**：基于多层感知机（MLP）的PPO策略网络

### 硬核模式
- **训练步数**：500万步
- **环境**：高难度 `BipedalWalkerHardcore-v3` 环境
- **训练优化技术**：与标准模式一致，同时适配更复杂的地形与更长的训练周期

训练基于 Stable Baselines3 库的PPO算法实现，并通过向量化环境并行运行多个实例，大幅提升训练效率。

## 3. 环境配置
环境配置通过 `env_utils.py` 中的两个核心函数实现：

### 3.1 `make_env()` 函数
`make_env()` 函数用于创建并配置训练/评估环境，支持多种可配置选项：
- **环境创建**：默认创建 `BipedalWalker-v3` 环境；设置 `hardcore=True` 可切换为 `BipedalWalkerHardcore-v3` 硬核模式。
- **渲染模式**：支持 `human`（实时可视化）和 `rgb_array`（视频录制）两种渲染模式。
- **视频录制**：设置 `record_video=True` 时，每1000步自动录制一次训练过程，视频文件将保存至指定目录。
- **监控器包装**：可通过监控器记录奖励、回合长度等性能指标，便于后续训练过程分析。
- **向量化环境**：使用 `DummyVecEnv` 实现多环境并行处理，加速训练流程。
- **观测与奖励归一化**：通过 `VecNormalize` 包装器对观测值和奖励值进行归一化，稳定训练过程，提升智能体学习效率。
- **帧堆叠**：默认堆叠最近4帧观测数据，为智能体提供时间上下文信息，这对理解运动动态至关重要。
- **观测值裁剪**：可设置 `clip_obs` 参数（默认值为10.0）限制观测值范围，避免训练过程中出现异常值干扰。

**示例用法：**
```python
env = make_env(env_name="BipedalWalker-v3", hardcore=True, record_video=True, use_monitor=True)
```

### 3.2 `observe_model()` 函数
`observe_model()` 函数用于加载训练好的PPO模型，并在指定环境中进行评估，同时自动适配训练时使用的 `VecNormalize` 和 `VecFrameStack` 包装器。

- **模型加载**：从指定路径加载训练完成的模型文件。
- **环境配置**：根据是否启用硬核模式，自动选择 `BipedalWalker-v3` 或 `BipedalWalkerHardcore-v3` 环境。
- **包装器适配**：若训练时使用了 `VecNormalize` 和 `VecFrameStack`，评估环境会自动应用对应配置，确保评估行为与训练过程一致。
- **评估过程**：在指定回合数内运行模型，返回平均奖励和奖励标准差作为评估结果。

**示例用法：**
```python
mean_reward, std_reward = observe_model(model_path='models/ppo_bipedalwalker_1M', n_eval_episodes=5, hardcore=False)
```

该配置确保了环境在训练和评估阶段的一致性，同时支持视频录制、奖励归一化等高级功能，兼顾了训练效率与评估的可扩展性。

## 4. 模型评估
模型评估通过 `observe_model()` 函数完成，该函数加载训练好的模型，并以 `human` 模式渲染运行过程，实现可视化评估。

### 评估结果示例：
- **标准模式**：平均奖励 `248.39 ± 112.10`
- **硬核模式（300万步训练）**：平均奖励 `-28.23 ± 24.82`
- **硬核模式（500万步训练）**：平均奖励 `-10.66 ± 3.91`
- **硬核模式（700万步训练）**：平均奖励 `-5.45 ± 2.10`

结果显示，智能体在标准环境中表现良好，但在硬核模式中仍存在明显性能瓶颈，需要进一步的训练或参数调优来提升表现。

## 5. 训练日志与分析
对500万步硬核模式训练的日志进行分析，可获得以下关键结论：
- **奖励变化趋势**：奖励值虽存在波动，但整体随训练推进逐渐趋于稳定。
- **回合长度变化趋势**：随着训练进行，智能体能够在环境中存活的时间不断延长，仅在少数阶段出现性能回落。
- **相关性分析**：奖励值与回合长度呈现显著正相关（相关系数0.89），说明智能体存活时间越长，获得的奖励越高。

项目使用 `pandas` 和 `matplotlib` 对奖励趋势、回合长度移动平均值等数据进行可视化，便于直观分析训练过程。

## 6. 改进方向
针对当前模型性能，提出以下优化建议：
- **调整学习率**：降低学习率可减少训练过程中的震荡，提升性能提升的稳定性。
- **重构奖励函数**：调整奖励权重，引导智能体优先学习平衡与生存能力，而非单纯追求向前移动。
- **增强探索能力**：引入ε-贪心算法或好奇心驱动探索等方法，帮助智能体学习更多样化的运动策略。
- **延长训练周期**：增加训练步数，为智能体提供更多学习经验，进一步优化策略。
