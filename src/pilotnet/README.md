## 项目简介

PilotNet 是一个端到端自动驾驶系统，通过深度神经网络直接从原始图像学习驾驶控制策略。该项目实现了 NVIDIA 的经典 PilotNet 架构，支持在 CARLA 仿真环境中进行数据采集、模型训练和预测测试。

### 主要特性

- **端到端学习**：直接从摄像头图像学习驾驶控制
- **CARLA 集成**：支持在 CARLA 仿真环境中进行数据采集
- **实时可视化**：提供车辆轨迹显示和镜头跟随功能
- **训练可视化**：实时损失曲线显示，直观展示训练过程
- **详细监控**：实时显示车辆状态和控制参数
- **灵活配置**：支持自定义训练参数和图像分辨率

## 环境准备

### 必需软件

1. [Anaconda/Miniconda](https://conda.io/projects/conda/en/latest/user-guide/install/index.html)
2. [CARLA 仿真模拟器](http://carla.org)
3. Python 3.8
4. TensorFlow 2.x

### 硬件要求

- **CPU**：建议 4 核心以上
- **内存**：建议 8GB 以上
- **GPU**：支持 CUDA 的 NVIDIA GPU（可选，用于加速训练）
- **存储**：至少 10GB 可用空间（用于存储训练数据）

## 安装步骤

1. **克隆仓库**
   ```bash
   git clone https://github.com/vishalkrishnads/pilotnet.git
   cd pilotnet
   ```

2. **创建并激活 conda 环境**
   ```bash
   conda create -n "pilotnet" python=3.8.0
   conda activate pilotnet
   ```

3. **安装 TensorFlow**
   
   如果有支持 CUDA 的 GPU：
   ```bash
   pip install tensorflow-gpu
   ```
   
   如果没有 GPU：
   ```bash
   pip install tensorflow
   ```

4. **安装其他依赖**
   ```bash
   pip install -r requirements.txt
   ```

5. **启动 CARLA 服务器**
   
   确保 CARLA 仿真器在 localhost:2000 端口运行。

6. **运行应用**
   ```bash
   python main.py
   ```

### 功能详解

#### 1. 使用已有数据训练模型

从录制的驾驶数据中训练神经网络模型。

**操作步骤：**
1. 选择菜单选项 1
2. 设置训练参数：
   - Epochs：训练轮数（默认 50）
   - Batch size：批处理大小（默认 64）
   - 模型名称：保存的模型文件名
   - 图像尺寸：输入图像的宽度和高度（默认 160×120）
3. 训练过程中会自动弹出实时损失曲线图
4. 训练好的模型将保存在 `models/` 目录下

**实时训练可视化：**

训练过程中会自动弹出一个图形窗口，显示四个损失曲线：

```
┌─────────────────┐  ┌─────────────────┐
│   Total Loss    │  │ Steering Loss   │
│ 蓝: Training    │  │ 蓝: Training    │
│ 红: Validation  │  │ 红: Validation  │
│                 │  │                 │
│  5.0 ─┐         │  │  1.8 ─┐         │
│       │         │  │       │         │
│  0.5 ─╂───┐    │  │  0.2 ─╂───┐    │
│       └───┘    │  │       └───┘    │
│      0  10     │  │      0  10     │
└─────────────────┘  └─────────────────┘
┌─────────────────┐  ┌─────────────────┐
│  Throttle Loss  │  │   Brake Loss    │
│ 蓝: Training    │  │ 蓝: Training    │
│ 红: Validation  │  │ 红: Validation  │
└─────────────────┘  └─────────────────┘
```
- **Total Loss**: 总损失（模型整体预测误差）
- **Steering Angle Loss**: 转向角度的预测误差
- **Throttle Pressure Loss**: 油门压力的预测误差
- **Brake Pressure Loss**: 刹车压力的预测误差

**训练曲线解读：**
- **蓝色曲线**：训练集损失（模型在训练数据上的预测误差）
- **红色曲线**：验证集损失（模型在验证数据上的预测误差）
- **理想状态**：两条曲线都下降并趋于平稳，且相互接近
- **过拟合迹象**：训练损失继续下降，但验证损失开始上升

**训练回调功能：**
- **早停机制**：当验证损失连续5轮不改善时自动停止训练
- **模型检查点**：自动保存验证损失最低的最佳模型
- **学习率调度**：每10轮自动降低学习率20%

**注意事项：**
- 高分辨率图像（如 1920×1080）会消耗大量内存，建议从默认分辨率开始
- 确保有足够的训练数据，避免数据不足导致的训练失败

#### 2. 生成新的驾驶数据

在 CARLA 仿真环境中录制驾驶数据。

**操作步骤：**
1. 确保 CARLA 服务器正在运行
2. 选择菜单选项 2
3. 输入录制时间（分钟）
4. 选择是否启用可视化：
   - **启用可视化**：显示车辆轨迹和镜头跟随，提供详细的终端信息
   - **禁用可视化**：仅录制数据，性能更好
5. 等待录制完成

**可视化功能：**
- **镜头跟随**：相机自动跟随车辆，提供流畅的视角
- **轨迹显示**：在 CARLA 中绘制车辆行驶路径（绿色线条）
- **实时数据**：终端显示当前速度、控制参数、车辆位置等
- **统计数据**：显示平均速度、最高速度、控制参数统计等

## 神经网络架构

PilotNet 采用卷积神经网络架构，包含以下层级：

### 卷积层
- 5 层卷积网络提取图像特征
- 使用 ReLU 激活函数
- 逐步降低空间维度，增加特征深度

### 全连接层
- 4 层全连接网络
- 使用 Dropout 防止过拟合
- 输出 3 个控制参数

### 输出层
- **转向角度**：控制车辆转向
- **油门压力**：控制车辆加速
- **刹车压力**：控制车辆制动

## 常见问题

### 训练阶段

**Q: 训练时出现内存不足错误**
- A: 降低图像分辨率或减少批处理大小

**Q: 训练数据不足**
- A: 增加录制数据量或减少训练轮数

### 数据生成器

**Q: 无法连接 CARLA 服务器**
- A: 确保 CARLA 服务器在 localhost:2000 端口运行
- A: 检查防火墙设置，确保端口未被阻塞

**Q: WSL 环境下连接失败**
- A: 使用 `ping $(hostname).local` 获取宿主机 IP
- A: 修改 `main.py` 中的连接地址

**Q: 磁盘空间不足**
- A: 清理 `recordings/` 目录下的旧数据

### 预测阶段

**Q: 模型加载失败**
- A: 检查模型文件路径是否正确
- A: 确保模型文件完整且未损坏

**Q: 预测结果异常**
- A: 检查输入图像尺寸是否与训练时一致
- A: 确保模型训练充分

## 性能优化建议

1. **数据采集**
   - 启用可视化功能以获得更详细的监控信息
   - 根据需要调整录制时间和数据质量

2. **模型训练**
   - 从低分辨率开始测试，逐步提升
   - 使用 GPU 加速训练过程
   - 合理设置 batch size 以平衡内存使用和训练速度

3. **系统配置**
   - 确保有足够的磁盘空间存储训练数据
   - 关闭不必要的后台程序释放系统资源
