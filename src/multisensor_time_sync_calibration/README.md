# 多传感器时间同步与延迟标定演示

本项目实现了一个面向自动驾驶感知融合的时间同步优化模块。相机和激光雷达在真实系统中往往存在采样频率不同、传输延迟不同、时间戳不完全对齐的问题，如果直接把两个传感器的目标轨迹进行融合，会出现轨迹拖影、位置偏移和融合误差增大的现象。本项目通过构造可复现实验场景，模拟同一运动目标被相机和激光雷达异步观测的过程，并自动估计激光雷达相对相机的时间延迟。

## 优化内容

- 构造二维曲线运动目标，生成带噪声的相机轨迹和激光雷达轨迹。
- 设置相机 12 Hz、激光雷达 9 Hz 的异步采样频率，模拟真实多传感器采样差异。
- 在激光雷达轨迹中加入 0.42 s 的时间戳延迟，形成可验证的标定目标。
- 使用公共时间轴插值，将不同采样率的观测数据转换到同一比较窗口。
- 对候选时间偏移进行网格搜索，计算相机与激光雷达轨迹之间的对齐残差。
- 自动选择残差最小的时间偏移作为估计延迟，并进行延迟修正。
- 将修正后的激光雷达轨迹与相机轨迹进行加权融合，输出融合轨迹。
- 对同步前后的融合轨迹分别计算相对真实轨迹的 RMSE，用量化指标证明优化效果。
- 输出三张可视化图和一份指标文件，方便提交文档和课堂说明。

## 运行方法

在本目录执行：

```bash
python main.py
```

也可以指定输出目录：

```bash
python main.py --output assets
```

运行后会在 `assets` 目录生成：

- `trajectory_alignment.png`：真实轨迹、原始传感器观测、同步后轨迹和融合轨迹对比。
- `offset_search_curve.png`：候选延迟与对齐误差曲线，展示算法如何找到最优延迟。
- `rmse_comparison.png`：同步前后融合误差对比柱状图。
- `offset_search.csv`：每个候选延迟对应的残差数据。
- `metrics.json`：估计延迟、误差和优化百分比等汇总指标。

## 实验结果

默认随机种子下，系统能够把激光雷达的 0.42 s 延迟估计到约 0.41 s，延迟误差约为 0.01 s。轨迹融合 RMSE 从同步前的较高误差下降到同步后的低误差，说明时间戳标定能够明显改善多传感器融合质量。

这个优化适合扩展到机器人、自动驾驶、无人机集群等场景。例如多个传感器上传目标坐标时，只要存在固定时间差，就可以先进行延迟估计，再做融合、跟踪和路径规划，避免系统把“时间不同步”误判为“目标位置不一致”。

## 测试

```bash
python -m pytest tests
```

测试内容包括：

- 延迟估计是否接近真实延迟。
- 同步后的融合误差是否低于同步前。
- 运行入口是否能生成指标文件和可视化图片。
